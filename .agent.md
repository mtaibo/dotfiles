# Agent context — dotfiles

This is a **Nix flake** dotfiles repo for Miguel Taibo, a first-year CS student.
It manages 3 machines from a single config: **MacBook** (macOS, aarch64), **desktop** (NixOS, x86_64), and **Raspberry Pi** (NixOS, aarch64).

## Quick reference

- **Entry point**: `flake.nix` — defines 3 outputs: `nixosConfigurations.nixos`, `darwinConfigurations.macbook`, `homeConfigurations.migueltaibo`
- **Machine configs**: `hosts/desktop/` (NixOS), `hosts/macbook/` (macOS + home-manager overrides)
- **Shared user config**: `modules/home/default.nix` — imports shell, packages, editors, browsers
- **Asset files** (raw configs): `assets/` — these are symlinked by Home Manager to `~/.config/`
- **NixOS system modules**: `modules/nixos/` — boot, networking, Hyprland, NVIDIA
- **macOS extras** in `hosts/macbook/home.nix`: fonts, Automator service for fullscreen Kitty (Cmd+K), kitty icon replacement, cyberpunk wallpaper via osascript

## Key architecture

```
flake.nix
├── nixosConfigurations.nixos → hosts/desktop/default.nix
│   ├── imports modules/nixos/ (boot, networking, hyprland, nvidia)
│   └── home-manager → modules/home/default.nix
├── darwinConfigurations.macbook → hosts/macbook/default.nix
│   └── home-manager → hosts/macbook/home.nix
│       └── imports modules/home/
└── homeConfigurations.migueltaibo → hosts/macbook/home.nix (standalone)
```

## Dotfile deployment mechanism

All config files in `assets/` are deployed via Home Manager's `home.file` mechanism (symlinks from Nix store to `~/.config/`). Firefox CSS is inlined (not symlinked). macOS activation scripts run imperatively (cp, osascript, PlistBuddy) to set wallpaper, replace app icons, and register keyboard shortcuts.

## Notes for the agent

- The repo uses **Nix language** (not YAML/JSON for config). All .nix files are Nix expressions.
- `pkgs.stdenv.hostPlatform.isLinux` and `isDarwin` are used for conditional config.
- `pkgsUnstable` is passed as a specialArg from flake.nix for access to unstable packages.
- `lib.mkForce` is used in macOS home.nix to override `homeDirectory` from the shared config.
- `lib.hm.dag.entryAfter` schedules macOS activation scripts in the right phase.
- wallpaper-grub.png was moved from `modules/nixos/` to `assets/grub/`.
- The original `modules/home/dotfiles/` was flattened into `assets/`.
- Nix modules were organized into subdirectories: `editors/`, `browsers/`.
- `hardware-configuration.nix` is auto-generated by nixos-generate-config — do not edit manually.
